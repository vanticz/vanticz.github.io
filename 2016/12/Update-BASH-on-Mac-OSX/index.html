Solution

To use a newer version of bash on Mac OSX, we can use homebrew. Execute the following command in Terminal to install Homebrew.

Once Homebrew is fully installed, install bash with Homebrew

After that, $ which bash would result in /usr/local/bin/bash instead of the previous /bin/bash. However, case.sh still doesn’t work.

Check bash version, it’s still 3.x:

Because the first line of version.sh still points to the old bash, fix this by changing it to the new bash:

Further settings

1.Allow Terminal to use the new shell, add it to the list of allowed shells:

$ sudo echo "$(brew --prefix)/bin/bash" >> /etc/shells

or $ sudo echo "/usr/local/bin/bash" >> /etc/shells

root password is required.

2.Change the login shell to the new bash:

$ chsh -s $(brew --prefix)/bin/bash

or $ chsh -s /usr/local/bin/bash

again root password is required.

Reopen Terminal and you are ready to bash 4.x!

Note: make sure /usr/local/bin and /usr/local/sbin are in your $PATH variable before /usr/bin and /usr/sbin
