Manacher's Algorithm

By Zeming Zhou

Published 2016-11-29

In computer science, the longest palindromic substring or longest symmetric factor problem is the problem of finding a maximum-length contiguous substring of a given string that is also a palindrome.

A palindrome is a word, phrase, number, or other sequence of characters which reads the same backward or forward, such as ‘madam’ or ‘kayak’.

Manacher’s algorithm can find the longest palindromic substring in time complexity O(n).

Algorithms are different when detecting odd-length palindrome and even-length palindrome. So the first step of Manacher’s algorithm uses a trick to eliminate this difference: pick a special character that does not exist in the original string, insert it on both sides of each character.

For example:

aba and | becomes |a|b|a|

abba and # becomes #a#b#b#a#

Now the length of the new string is guaranteed to be odd, we call this new string S for simplicity.

Then it uses an helper array/list P to track the radius of the longest palindrome centered at each character. P[i] is the radius of the longest palindrome centered at S[i], and P[i] has a minimum value 1, when the longest palindrome is S[i] itself.

For example:

We can prove that P[i]-1 is the length of palindrome centered at S[i] in the original string.

Proof:

A palindrome in the new string S must begin and end with #

e.g. #, #a#, #a#b#b#a#

If we get rid of a # either at the begining or the end of palindrome, each character has a # to pair with.

e.g. (), (a#), (a#)(b#)(b#)(a#)

That means the length of palindrome in S minus one is twice the size of palindrome in the original string.

Let L = 2 * P[i] - 1 be the length of palindrome in S centered at S[i], then the length of palindrome in original string is (L - 1) / 2,
equals to P[i] - 1.

End of proof.

Because palindromics are symmetric, we can use dynamic programming to reuse some calculation. We add aother different special character at the begining to prevent potential index out of bound error, now useful indices starts from 1.

Using Document Structure

Understanding parent-child relationship:

HTML documents are based on hierarchy of elements, each element is either the parent or the child of another element (usually both, except leaves and the root ). Generally in tree structure of HTML document, elements above one element is called ancestor, and elements beneath it are called descendant. Parent and child are specific terms, they are ancestor or descendant of exactly one layer.

Descendant Selectors: (AKA, contextual selectors) a white space between selectors. Example: h1 em {color: gray;} This will ‘gray’ any em elements that is a decendant of an h1.

Selecting Children: using the greater-than sign, Example: h1 > strong {color: red;} This selects the first bot not the second line below.

In the first line, strong is a child of h1, while in the second line, strong is a descendant of h1, not a child.

Selecting Adjacent Sibling Elements: using the plus sign. Example: h1 + p {margin-top: 0}; This selects the paragraph “immediately follows an h1 element that shares a parent with the p element”, can not select the sibling before.

Pseudo-Classes and Pseudo-Elements

Syntax: Pseudo- started selectors share the same syntas, the colon.

Pseudo-Class Selectors:

  • Link pseudo-classes
    1. :link is for unvisited link, can be written as a:link {clolr: blue;} but this is redundant, usually just omitted. a {clolr: blue;}
    2. :visited for hyperlinks to already visited address. a:visited {clolr: red;}
  • Dynamic pseudo-classes
    1. :focus is the element currently has the input focus, example input:focus {...}
    2. :hover is the element your mouse pointer is placed on.
    3. :active is the element activated by user input, like click and hold mouse on an element.
  • Selecting a first child
    • This could be misunderstanding, for example p:first-child DOES NOT select the first child of a p element, it selects any p element that is the first child of another element.
  • Selecting based on language
    • *:lang(en) {...} this is exactly like |= attribute selector from last post.

Combining pseudo-classes is just like grouping, example: a:link:hover:lang(en) {...}

Pseudo-Element Selectors:

  • :first-letter and :first-line are literally the first letter and line of an element.

  • :before and :after: Used to add contents before or after an element, for example, the following code add “[[“ and “]]” around any h2 elements.

CSS-Selectors

By Zeming Zhou

Published 2015-11-10

To begin with

I’m following the book Cascading Style Sheets: The Definitive Guide for posts with prefix “CSS-“. I chose this book because other than simply introducing syntax and basics, it digs deep what CSS can do, and I personally like O’Reilly Media series.

Now let’s start with today’s topic: Selectors

Quick review on basic CSS selectors

Element Selector: Selector {Property: Value; Property: Value;}

Universal Selector: * {Property: Value;}

Class Selector: .Classname {Property: Value;}

ID Selector: #IDname {Property: Value;}

notice: Class is literally for elements sharing same properties, you may assign classes to any number of elemnts. While ID are used only once within an HTML document.

Although browser don’t usually check uniqueness of IDs in HTML, meaning you can use same ID multiple times. This could make DOM scripting getElementById() more difficult so it’s not recommended, use a Class instead works better.

Class and ID are case-sensitive.

Attribute Selectors

Basic Attribute Selector: h1[class] {color:red'}, this will make all text below red

Selection Based on Exact Attribute Value: Add one value for attribute in []. For example, h1[class="hello"] {color:red'} will only make the “Hello” above red but leave other two words unchanged.

Selection Based on Partial Attribute Values: “For attributes with space-separated list of words, use [attr~="val"] to select based on presence of any one of those words”. For example, span[class~="windy"] {font-style: italic;} will make Vancover and Montreal italic, and Toronto untouhed. (Please ignore poor word choice)

Here’s more partial-value attribute selectors using Regular Expression

A Particular Attribute Selection Type: *[lang|="en"] {color: white;} selects any element whose lang attribute is equal to en or begins with en-. This is useful when you have a series of, say, image files figure-1.jpg, figure-2.png, figure-1.gif, etc. You can select the all using img[src|="figure"] {}
